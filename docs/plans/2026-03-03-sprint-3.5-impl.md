### Task 1: Audit Tamil translations against style guide

**Files:**
- Modify: `frontend/messages/ta.json`
- Reference: Tamil style guide in memory (`tamil-style-guide.md`)

**Step 1: Review all Tamil strings for grammar violations**

Audit each string in `ta.json` against the Tamil style guide rules. Key checks:

1. **Vallinam doubling** (CAT3): After அந்த/இந்த/எந்த, infinitives, adverbs, -ஆக suffix, accusative (ஐ), dative (கு), compounds
   - e.g. "இந்த + பக்கத்தை" → needs doubling check
   - e.g. "பதிவு + செய்" after infinitive → vallinam check

2. **JOIN vs SEPARATE** (CAT1/CAT2):
   - Noun (no case) + postposition → JOIN (e.g. இதுபற்றி)
   - Noun WITH case + postposition → SEPARATE (e.g. இவற்றைப் பற்றி)
   - Word + வேண்டும்/முடியும்/கூடாது → SEPARATE
   - Word + கூடிய/கூடும் → JOIN
   - Participial adj + போது/வரை → JOIN

3. **Natural phrasing**: Not machine-translated from English/Malay

4. **Consistency**: Same Tamil term used for the same English concept throughout

**Step 2: Apply fixes to `ta.json`**

Edit each violation found. Document what was changed and why.

**Step 3: Run translation completeness test**

Run: `cd frontend && npx jest --testPathPattern translations --forceExit --no-coverage`
Expected: PASS — all three language files have matching keys.

**Step 4: Run full frontend tests**

Run: `cd frontend && NODE_OPTIONS="--max-old-space-size=512" npx jest --forceExit --no-coverage`
Expected: 215 tests passing.

**Step 5: Commit**

### Task 2: Deploy backend to production

**Step 1: Verify gcloud config**

```bash
gcloud config get account
gcloud config get project
```
Expected: `admin@tamilfoundation.org` and `sjktconnect`. If not:
```bash
gcloud config set account admin@tamilfoundation.org
gcloud config set project sjktconnect
```

**Step 2: Deploy backend**

```bash
cd backend && gcloud run deploy sjktconnect-api --source . --region asia-southeast1 --allow-unauthenticated
```
Expected: Successful deployment with new revision URL.

**Step 3: Verify health check**

```bash
curl https://sjktconnect-api-748286712183.asia-southeast1.run.app/health/
```
Expected: `{"status": "ok"}`

**Step 4: Verify national stats endpoint**

