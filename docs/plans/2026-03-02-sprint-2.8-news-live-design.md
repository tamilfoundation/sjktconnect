# Sprint 2.8 Design — News Watch Goes Live

**Date**: 2 March 2026
**Sprint**: 2.8 (final Phase 2 sprint)
**Status**: Approved

---

## Objective

Expose approved news articles to the public frontend and automate the daily pipeline, completing Phase 2.

## Deliverables

### 1. Public News API

**Endpoint**: `GET /api/v1/news/`

- Public (no auth required)
- Only returns articles where `review_status = APPROVED`
- Supports filters:
  - `?school=<moe_code>` — articles mentioning a specific school (JSON contains lookup)
  - `?is_urgent=true` — urgent articles only
- Paginated (default 10, max 50)
- Response fields: id, title, source_name, published_date, ai_summary, sentiment, relevance_score, mentioned_schools, is_urgent, url

**Implementation**: New serializer + viewset in `newswatch/api/`, registered in project URLs.

### 2. Real NewsWatchSection on School Pages

Replace the placeholder `NewsWatchSection` component with a real implementation:

- Accepts `moeCode` prop
- Fetches from `/api/v1/news/?school=<moe_code>`
- Displays: headline (links to original article), source name, date, AI summary
- Sentiment badge (colour-coded)
- Urgency indicator for flagged articles
- "No news articles yet" fallback
- Loading skeleton state

### 3. Cloud Scheduler Automation

Two new Cloud Run Jobs + Schedulers:

| Job | Schedule | Command |
|-----|----------|---------|
| `sjktconnect-news-pipeline` | Daily 8:30 AM MYT | `fetch_news_alerts && extract_articles && analyse_news_articles` |
| `sjktconnect-monthly-blast` | 1st of month, 9:00 AM MYT | `compose_monthly_blast` |

Uses the same pattern as the existing `sjktconnect-check-hansards` job.

## Out of Scope

- Standalone `/news/` listing page (Phase 3)
- Individual article detail pages (Phase 3)
- Search functionality (Phase 3)
- Magic Link gating (news is public)
- Relational FK between NewsArticle and School (JSON field sufficient for now)
- News articles not linked to any school (general Tamil school news — Phase 3)

## Technical Notes

- `mentioned_schools` is a JSONField containing school names/codes extracted by AI — filtering uses Django's `__contains` lookup
- The school-to-article link is soft (no FK) — acceptable for MVP but worth revisiting in Phase 3
- Frontend API function added to `lib/api.ts`, TypeScript type added to `lib/types.ts`

## Deploy

- Backend: Cloud Run (`sjktconnect-api`)
- Frontend: Cloud Run (`sjktconnect-web`)
- Cloud Scheduler: `gcloud scheduler jobs create http` (same pattern as existing Hansard job)
